Tier A — Prose Passes

Scope by command

Triggered by prompt scope, not document structure. Run before any Tier B work on the same content. All eight sub-passes run together unless you call one individually.

CommandSub-passWhat it covers
"Prose pass [scope]"Full Tier AAll eight sub-passes on the stated scope (e.g. Ch 5, Chs 1–4, book-wide).
"Word-by-word [scope]"A1Duplicate words, comma splices, doubled punctuation, missing words.
"Spelling check [scope]"A2Spelling, grammar, capitalization, smart quotes, em-dash audit, institutional singular, Over-classified, Government.
"AI-speak [scope]"A3Watch-list scan. Choice form per hit: Keep / Use proposed / Write my own.
"Paragraph logic [scope]"A4Each paragraph opens, develops one idea, closes cleanly. Flags over-150-word paragraphs and parallel openers.
"Paragraph structure [scope]"A5Too-short paragraphs (≤2 sentences, can't stand alone); over-long without natural break.
"Flow check [scope]"A6Prohibited patterns: em-dashes in body, throat-clearing, Not-X constructions, passive voice, bullet lists. Voice, register, rhythm.
"Thematic echo [scope]"A7Repeated images, phrases, or structural moves within a chapter or across adjacent chapters.
"Sterling [scope]"A8Chapter title weight, pull quote differentiation, connective tissue, tonal register. Advisory notes unless significant issue found.

Tier B — Mechanical Passes

Scope by command — each logs on completion

Run after Tier A is confirmed clean for the same content. Each sub-pass is discrete and writes a timestamped log entry before packing. If content is edited after a log entry, that passage is flagged for re-check at next session open.

CommandSub-passWhat it covers
"Formatting pass [scope]"B1Style ID compliance, image placement rule, caption construction, section breaks, pull quote/Attribution pairs, TextOpen after subheadings, chapter structure order, two-space title normalization.
"Citation pass [scope]"B2Four-point check: (1) every significant factual claim has a marker; (2) all citations are primary sources per hierarchy; (3) each citation is fetched and verified to support the specific claim; (4) every citation appears in A15 Bibliography with a complete entry. Logs to Citation_Verification_Log.md.
"Cross-ref pass [scope]"B3Every "see Chapter N" and "Appendix AX" reference verified to resolve to real content with matching material. Back matter included unless Dave specifies otherwise. Logs to CrossRef_Verification_Log.md.
"Footnote pass [scope]"B4Four-count gate (ref = note bookmarks = forward = return hyperlinks). Adjacent marker audit. Sequence integrity 1→N. afootnote construction check. No orphaned markers.
"Appendix pass"B5A2 sightings scan, A12 figures/diagrams, A15 Quotations location codes, A15 Bibliography new entries, Glossary new terms, Index new subjects. Run after B2 is confirmed for relevant chapters.
"URL pass"B6Every hyperlink verified to resolve. Invisible-hyperlink rendering confirmed (color=auto, u=none, no Hyperlink style).
"Index pass"B7Run last — after all content is locked. Locator codes, bookmark naming, subentry formatting, print vs. Kindle locator styles, exclusion rules (A15, A12). Editing body content after B7 potentially invalidates locators.

Help & Settings

About This App

This app is designed to make project management an effective tool for working side by side with Claude. It gives you a structured, persistent workspace where notes, roadmap phases, research, outreach contacts, and reference materials all live in one place — synchronized with Claude at the start and end of every session.

It runs entirely in your browser as a single HTML5 file. There is no server, no login, and no subscription. Your data lives in localStorage on your device and in a GitHub repository that Claude reads and writes directly.

Functions

Click a tab above to get more information on how to use each function.

Roadmap
Gantt-style project timeline. Organize work into phases, tasks, and stage gates with target dates. Drag to reorder, click to edit, track completion.
Notes
Project notes, questions, reminders, and dialogue threads. Filter by type, tag, or status. Flow items to Claude as conversation starters.
Resources
Links, documents, and reference materials organized by tag. Attach URLs, descriptions, and source notes for quick retrieval.
Outreach
Contact management for blurb requests, press outreach, agent queries, and publisher submissions. Track status, notes, and follow-up dates.
Reference
Quick-access panels for production passes, style guide, ISBNs, pricing, and distribution details. Read-only reference material.
Settings
Customize project name, tags, note types, and sync preferences. Configure how the hub behaves across devices and sessions.

Devices & Requirements

This is an open-platform HTML5 application with JavaScript. JavaScript must be enabled in your browser for the app to function. It has been specifically tested on:

Mac — Safari 17+, Chrome 120+
 📱iPad — Safari (iPadOS 16+). Touch drag for roadmap phase reordering supported.
 📱iPhone — Safari (iOS 16+). Full read/write; some Gantt interactions easier on larger screens.
🖥Windows / Linux — Chrome or Firefox. Not primary test platforms; known to work.

Local storage is per-browser per-device. Use Update Claude to sync across devices via GitHub.

Roadmap

The Roadmap is a Gantt-style view of your project timeline. It organizes work into phases, each containing tasks and optional gates (decision or approval checkpoints).

Navigating the screen

  • Gantt scroll: Two-finger trackpad swipe (Mac) or two-finger drag (iPad).
  • Open / collapse a phase: Tap the phase header to expand or hide its tasks.
  • Gate diamonds: Click a gate diamond to mark it met (filled) or unmet (outline).
  • Column resize: Drag the handle at the right edge of the phase label column.

Adding and editing

  • Use the + Phase and + Task buttons to add items.
  • Click any task bar or phase label to edit dates, notes, and completion status.
  • Drag phases up or down to reorder (iPad: touch-drag supported).

Keyboard shortcuts

  • ↑ / ↓ — Scroll the Gantt view up or down one row.
  • ← / → — Scroll the Gantt timeline left or right.
  • + — Expand all phases (show all tasks).
  • − — Collapse all phases (show phase headers only).

Notes

Notes is your project journal — a filterable list of items in several types.

Item types

  • Project Note — general observations, decisions, or reference entries.
  • Question — open questions awaiting answers or research.
  • Reminder — date-linked to-do items. Appear in the Reminders filter. Only open, non-deferred reminders count toward the Reminders badge.
  • For Claude / Dave — dialogue threads. Toggle To Claude when adding to start a thread.
  • Roadmap — mirror of all roadmap phases, tasks, and gates. Filterable and expandable inline. Roadmap items are not included in the Open, Done, Deferred, Priority, or Reminders counts — those counts reflect Notes-only items. The Roadmap type filter shows roadmap items with their own count from the Roadmap tab.
  • Custom types — add your own types (e.g. Research) in Settings → Note Types. Custom types appear in all type selectors and the filter dropdown. Built-in types cannot be removed.

Filters

  • Filter by status (Open, Deferred, Done, Priority) using the chips at the top. All = open + deferred + done; Priority and Reminders are subsets and not added separately.
  • Filter by tag or type using the dropdowns on the right. Each type shows its open item count.
  • The Reminders chip filters to open reminder-type items only and stacks with the status filter.
  • Reset clears all filters and returns to the default Open view.

Item overlay

  • Click any item to open the full-screen overlay showing the item text, action buttons, and thread.
  • Long item text is truncated — click ▼ More to expand, ▲ Less to collapse.
  • Use Dave reply / Plain note toggle to set the entry type before sending.
  • Thread entries are banded: yellow = Dave, blue = Claude, white = plain note. Newest first.
  • Click Edit in the action bar to edit all fields inline — item text, type, tag, dates, link, answer.
  • Right-click any collapsed card for a quick context menu: Open, Edit, Mark done, Defer, Delete.

Keyboard shortcuts

  • ↑ / ↓ — Move selection up or down through the visible item list.
  • Enter — Open the selected item's overlay.
  • + — Expand the selected item inline (show body).
  • − — Collapse the selected item.
  • E — Edit the selected item (opens overlay in edit mode).
  • D — Toggle done on the selected item.
  • Cmd/Ctrl+Enter — Save new item (cursor in note input) or send reply (cursor anywhere in the overlay reply area).
  • Cmd/Ctrl+Z — Undo the last action. Works anywhere — in input fields, the overlay, the roadmap, or while browsing. Up to 10 actions deep.
  • Cmd/Ctrl+Delete — Delete the selected item (confirm dialog appears).
  • Escape — Close the overlay, dismiss search results, or hide the undo toast.

Shortcuts E, D, +, −, Enter, and Cmd+Delete are inactive while typing in any input or textarea. Cmd+Z and Cmd+Enter always work.

Undo

  • Every data-changing action — mark done, mark open, defer, delete, edit, add item, reply, accept, roadmap step toggle — is undoable.
  • After any such action, a dark toast appears at the bottom of the screen for 8 seconds with an ↺ Undo button.
  • Cmd/Ctrl+Z also undoes the last action at any time, even after the toast fades.
  • The undo stack holds the last 10 actions. There is no redo.

Tags

  • Tag fields appear in the main input form, inline item edit, overlay edit mode, and the overlay reply form.
  • All tag fields share a single keyword list. Start typing to filter existing tags or type a new word.
  • To add a new tag: type it into any tag field and press Enter (in the overlay reply form) or navigate away. The new keyword is immediately added to the list and available everywhere.
  • Manage the full keyword list in Settings → Keywords: add in bulk (comma-separated) or remove individual tags.
  • In the overlay reply form, press Enter in the tag field to commit the tag and move focus to the reply textarea, then Cmd+Enter to send.

Resources

Resources stores links, documents, and reference materials relevant to your project. Each entry can have a URL, description, tag, and source note.

Adding resources

  • Use the + Resource form at the top of the tab.
  • Assign a tag to group related resources together.
  • Filter by tag using the tag selector.

Outreach

Outreach manages contacts for blurb requests, press outreach, agent queries, and publisher submissions. Each contact card tracks status, notes, materials sent, and follow-up dates.

Contact cards

  • Add a contact with name, role, and organization.
  • Track status: Not contacted, Sent, Responded, Declined, Committed.
  • Add timestamped notes to each card as the relationship develops.
  • Attach a materials link (press kit, sample chapter, etc.).

Reference

The Reference deck provides quick access to production and style information. Panels include Passes, Style & Chapter Map, ISBNs & Pricing, and Distribution. These are read-only reference materials updated by Claude at session end.

Settings

Settings lets you customize how the hub behaves for your project. Changes are saved to localStorage immediately and pushed to GitHub when you sync with Claude.

How sync works

The hub now uses a separate data.json file on GitHub as the single source of truth. Every time the hub loads on any device, it fetches the latest data.json from GitHub — so iPad, Mac, and any other device are always in sync automatically, with no manual intervention.

localStorage on your device acts as an instant-load cache. On startup the hub renders from localStorage immediately (no delay), then silently checks GitHub in the background. If GitHub has newer data, the UI updates automatically.

The Update Claude workflow

Press Update Claude to copy a JSON snapshot of your current data to the clipboard. Paste it into a Claude session — no additional prompt needed. Claude reads the snapshot, applies your changes, and pushes an updated data.json back to GitHub. The next time you load the hub on any device, it gets the new data automatically.

What Claude pushes

Claude pushes data.json only (~20KB) for data changes. The app file (index.html, ~300KB) is only updated when interface changes are made. This keeps pushes fast and separates your data from the app code.

Collision prevention

Avoid running two Claude sessions that both push to GitHub simultaneously. One session writing at a time is the safe pattern.
